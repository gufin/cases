## Why

Биллинговая система Юрайт.Суды должна иметь возможность обновлять подписку пользователей через вебхук и контролировать лимиты на количество отслеживаемых дел. Без этого невозможно монетизировать сервис: нет механизма обновления плана при оплате и нет защиты от превышения лимитов бесплатного тарифа.

## What Changes

- Новый эндпоинт `POST /api/billing/update` — принимает вебхук от биллингового сервиса, обновляет поля подписки пользователя (`plan`, `cases_limit`, `subscription_ends_at`, `billing_updated_at`) по `keycloakSub`. Защищён статичным токеном `BILLING_WEBHOOK_SECRET` в заголовке.
- Сервисный слой проверки лимитов — перед сохранением дела (`POST /api/cases/save`) проверяет, что текущее количество отслеживаемых дел пользователя не достигает его `cases_limit`. При превышении возвращает `HTTP 402 Payment Required`.
- Новая переменная окружения `BILLING_WEBHOOK_SECRET` добавляется в `core/config.py`.

## Capabilities

### New Capabilities

- `billing-webhook`: Эндпоинт `POST /api/billing/update` — обновление полей подписки пользователя по вебхуку с аутентификацией через статичный токен.
- `cases-limit-check`: Сервисный слой, который считает активные дела пользователя и сравнивает с `cases_limit`, возвращая `402` при достижении лимита.

## Impact

- **Новые файлы**: `apps/backend/app/api/v1/billing.py`, `apps/backend/app/services/billing.py`
- **Изменённые файлы**: `apps/backend/core/config.py` (новый параметр `BILLING_WEBHOOK_SECRET`), `apps/backend/main.py` (подключение нового роутера), `apps/backend/app/api/v1/cases.py` (вызов лимит-чека)
- **Зависимости**: уже добавлен `SQLAlchemy 2.0+`, `FastAPI`; новых зависимостей нет
- **БД**: только чтение/обновление существующей таблицы `users`; счётчик `casesUsed` вычисляется динамически через `COUNT` по таблице `cases` (колонка `is_watched = True`)
