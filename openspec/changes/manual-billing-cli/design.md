## Context

Таблица `users` уже содержит поля `plan`, `cases_limit`, `subscription_ends_at`, `billing_updated_at` (см. `user-model` spec). Платёжный хаб не запущен, поэтому нет автоматического пути обновить эти поля — только прямой SQL. CLI-скрипт устраняет эту проблему для этапа MVP.

Скрипт запускается внутри контейнера `backend`, где уже настроены `DATABASE_URL` и все зависимости, включая SQLAlchemy-модели.

## Goals / Non-Goals

**Goals:**
- Скрипт находит пользователя по `email` и обновляет его биллинговые поля
- Поддерживает `--plan`, `--limit`, `--days` (дней подписки от сейчас)
- Печатает итоговое состояние после изменений
- Корректно обрабатывает ошибки: пользователь не найден, недопустимый план

**Non-Goals:**
- Не добавляет HTTP-эндпоинт для биллинга (это задача платёжного хаба)
- Не реализует UI-интерфейс администратора
- Не валидирует платёжные данные (карты, транзакции)

## Decisions

**Поиск пользователя по email, не по keycloak_sub**
Email удобнее для администратора, чем UUID из Keycloak. Поле `email` уже присутствует в модели `User`. Риск коллизии минимален, так как пользователи вводят email уникальный на уровне Keycloak.

**Параметр `--days` вместо явной даты**
Администратору проще сказать «продлить на 30 дней», чем вводить ISO-дату. Скрипт сам вычисляет `subscription_ends_at = now() + timedelta(days=N)`. Если `--days 0` — поле обнуляется (`null` = бессрочно).

**Использование существующей SQLAlchemy-сессии**
Скрипт импортирует `SessionLocal` и модель `User` из уже существующего приложения. Это исключает дублирование конфига подключения к БД.

**argparse вместо Click**
Click не является обязательной зависимостью проекта. argparse входит в стандартную библиотеку Python и не требует установки пакетов.

## Risks / Trade-offs

- [Риск: опечатка в email] → Скрипт выведет "Пользователь не найден" и завершится с кодом 1. Изменений не произойдёт.
- [Риск: запуск не в том контейнере] → `DATABASE_URL` не будет доступен; SQLAlchemy выбросит ошибку подключения. Пользователю нужно убедиться, что команда запускается через `docker compose exec backend`.
- [Риск: `--days 0` как «обнуление» неочевиден] → Задокументировать в `--help` и в README.
